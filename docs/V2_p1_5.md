## 1.5. Technology Stack Selection

### 1.5.1. Technologies Carried Over from v1

- **Kotlin**: The main programming language. Provides modern syntax and type safety.
- **Android View System**: Traditional approach to building UI using XML markup. Retained for continuity and working with existing Activity and Fragment.
- **Room Persistence Library**: Android Jetpack component for creating and managing local SQLite database. Used for storing the list of resources, destinations, and settings.
- **ExoPlayer**: Google's media player for video and audio format playback. Chosen for flexibility, extensibility, and network stream support.

### 1.5.2. New Technologies for v2

- **Kotlin Coroutines & Flow**: Standard for managing asynchronous operations (network requests, DB access, file operations), ensuring UI responsiveness.
- **Android Architecture Components**:
  - **ViewModel**: For storing and managing UI-related data, surviving configuration changes.
  - **Navigation Component**: For implementing and standardizing navigation between screens, solving the problem of logical inconsistencies during transitions.
- **Hilt**: Google's recommended library for dependency injection. Simplifies dependency management and improves code testability.
- **smbj**: Modern Java library for working with SMBv2/v3 protocol. Necessary for accessing network folders in the local network.
- **sshj**: Library for implementing SFTP client, necessary for accessing SFTP resources.
- **Cloud Storage APIs**:
  - **Google Drive API**: For integration with Google Drive.
  - **Microsoft Graph API**: For integration with OneDrive.
  - **Dropbox API**: For integration with Dropbox.
